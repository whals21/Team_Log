## 우선 정화 권장 TOP 5

### 1. #25 데미지 공식 불일치 — CRITICAL
- **문제**: 적 공격이 방어력(DEF)을 무시함
- **위치**: `EnemyAIController.AttackTarget()` (line 131-134)
- **구현**: `TurnManager.ExecuteAttack()`은 `ATK + Power - DEF`이지만, `EnemyAIController.AttackTarget()`은 `ATK + damage` 후 `TakeDamage` 직접 호출
- **해결**: EnemyAIController에도 DEF 차감 로직 추가, 또는 TurnManager의 공식을 공통 메서드로 추출

### 2. #15 BattleSceneSetup 위치 — CRITICAL
- **문제**: UI 폴더(`UI/Battle/`)에 위치하지만 전체 전투 흐름을 제어하는 오케스트레이터
- **위치**: `Assets/02.Scripts/UI/Battle/BattleSceneSetup.cs`
- **해결**: `Assets/02.Scripts/Combat/`으로 이동, UI 초기화 로직만 BattleUISetup으로 분리

### 3. #7 EffectProcessor.cs — HIGH (죽은 코드)
- **문제**: 전체 파일이 어디에서도 호출되지 않음
- **위치**: `Assets/02.Scripts/Combat/StatusEffect/EffectProcessor.cs`
- **중복**: Character.cs가 동일 로직을 직접 수행 (Poison, Burn만 처리, Bleed/Regen 미처리)
- **해결**: 삭제 후 필요한 로직(Character에 없는 Bleed, Regeneration 등)은 Character.cs로 이관

### 4. #9 BattleEventManager — HIGH (미사용 이벤트 버스)
- **문제**: 싱글톤 이벤트 버스가 이벤트를 발생시키지만 구독자가 없음
- **위치**: `Assets/02.Scripts/UI/Battle/BattleEventManager.cs`
- **실상**: UI→로직 통신이 직접 참조(`_turnManager.ConfirmActions()` 등)로 이루어짐
- **해결**: 이벤트 버스를 실제로 활성화하거나, 직접 참조를 이벤트로 전환

### 5. #10 TestCombatManager.cs — MEDIUM (중복 테스트 하네스)

- **해결**: BattleSceneSetup으로 통합 후 삭제

## 상세 분석

### 데미지 공식 불일치 상세 (#25)

**플레이어 공격 (정상)**:
```csharp
// TurnManager.ExecuteAttack() - line 132-137
int damage = caster.Stats.GetStat(StatType.ATK) + skill.Power;
int defense = target.Stats.GetStat(StatType.DEF);
int finalDamage = System.Math.Max(1, damage - defense);
target.Health.TakeDamage(finalDamage);
```

**적 공격 (버그)**:
```csharp
// EnemyAIController.AttackTarget() - line 131-134
private void AttackTarget(Character target, int damage)
{
    int finalDamage = _owner.Stats.GetStat(StatType.ATK) + damage;
    target.Health.TakeDamage(finalDamage);  // ← DEF 차감 없음!
}
```

**영향**: 적의 공격이 항상 최대 데미지를 줌. 예: 고블린 ATK 12 + 공격값 10 = 22 고정 데미지 (DEF 8 무시)

### 상태이상 처리 이중 경로 상세 (#26)

**Character.cs (실제 사용됨, 일부 처리)**:
- Poison: DoT 적용 O
- Burn: DoT 적용 O
- Bleed: 처리 X (버그)
- Regeneration: 처리 X (버그)
- Stun/Freeze/Sleep: 행동 불가 처리 X (주석만 있음)

**EffectProcessor.cs (미사용, 전체 처리)**:
- Poison, Burn, Bleed, Regeneration: 모두 처리 로직 존재
- Stun, Freeze, Sleep: ProcessTurnStart에 로직 있지만 미호출
- Freeze: +50% 피해 증가, Shield: 피해 흡수, Bleed: 치유 -50%

**결과**: Bleed, Regeneration, Shield, Freeze 피해 증가, Sleep 등의 상태이상이 데이터로 정의되어 있으나 실제로는 작동하지 않음
