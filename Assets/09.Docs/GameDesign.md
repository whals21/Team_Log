# Team Log - Game Design Document

## 1. 게임 개요

| 항목 | 내용 |
|------|------|
| **장르** | 턴제 전투 로그라이크 |
| **플랫폼** | PC (Unity) |
| **플레이어** | 1인 (4인 파티 조작) |
| **핵심 컨셉** | 스킬 선택이 아닌 랜덤 스킬 드로우 |

### 핵심 아이디어
- 매 턴, 각 캐릭터는 보유한 4개 스킬 중 **1개를 랜덤**으로 뽑아 사용
- 플레이어는 "어떤 스킬을 쓸지"가 아니라 **"랜덤으로 주어진 스킬을 어떻게 조합할지"** 선택
- 전략의 핵심: 상황에 맞는 캐릭터 순서 배치 + 스킬 타겟팅

## 2. 전투 시스템

### 2.1 턴 구조

```
[전투 시작]
    ↓
[턴 시작]
    ↓
[드로우 페이즈] → 각 캐릭터별로 스킬 1개 랜덤 드로우
    ↓
[액션 페이즈] → 플레이어가 순서/타겟 지정
    ↓
[실행 페이즈] → 지정된 순서대로 스킬 실행
    ↓
[적 턴] → 적 행동
    ↓
[턴 종료] → 다음 턴 시작
```

### 2.2 스킬 드로우 시스템

```
캐릭터당 스킬 슬롯: 4개
    ↓
턴 시작 시 각 캐릭터마다 1개 스킬 랜덤 선택
    ↓
플레이어에게 4개의 스킬 카드 제공 (캐릭터별 1개씩)
    ↓
플레이어가 실행 순서와 타겟 지정
```

## 3. 캐릭터 시스템

### 3.1 기본 스탯

| 스탯 | 설명 |
|------|------|
| HP | 체력 |
| ATK | 공격력 |
| DEF | 방어력 |

### 3.2 역할 클래스 (예시)

| 클래스 | 특징 | 스킬 예시 |
|--------|------|-----------|
| **전사** | 높은 HP, 근접 딜러 | 강타, 방패, 도발, 분노 |
| **마법사** | 높은 ATK, 낮은 HP | 파이어볼, 얼음창, 마나방어, 메테오 |
| **힐러** | 치유, 버프 | 치유, 보호막, 정화, 부활 |
| **도적** | 높은 SPD, 치명타 | 급소공격, 은신, 독바르기, 연막 |

### 3.3 스킬 구조

```
Skill
├── Name: 스킬 이름
├── Description: 설명
├── Type: 공격/치유/버프/디버프
├── TargetType: 단일/전체/아군/자신
├── Power: 위력 (기본 수치)
├── Cost: 비용 (있다면)
├── Effects: 추가 효과 (상태이상 등)
└── Weight: 드로우 가중치
```

## 4. 적 시스템

### 4.1 적 행동 패턴

- **패턴 기반**: 미리 정해진 행동 순서
- **의도 표시**: 다음 턴에 할 행동을 플레이어에게 미리 공개
  - 예: "공격 준비", "버프", "강력한 공격 충전"

### 4.2 적 타입

| 타입 | 특징 |
|------|------|
| 일반 | 기본 스탯, 단순 패턴 |
| 엘리트 | 높은 스탯, 특수 능력 |
| 보스 | 다중 페이즈, 특수 메커니즘 |

## 6. 로그라이크 맵 시스템 (Phase 3)

### 6.1 맵 구조

- **층(Floor)**: 총 3층으로 구성. 각 층마다 독립 맵 생성
- **노드(Node)**: 플레이어가 방문할 수 있는 각 위치
- **분기(Branch)**: Slay the Spire식 상향 분기 구조

```
층 구성:
┌──────────────────────────────────────────────────┐
│ 층 1 (3단계)  시작 → 전투 → 전투/이벤트 → 보스   │
│ 층 2 (4단계)  시작 → 전투 → 상점/이벤트 → 엘리트 → 보스 │
│ 층 3 (5단계)  시작 → 전투 → 이벤트 → 엘리트 → 상점 → 최종보스 │
└──────────────────────────────────────────────────┘
```

### 6.2 노드 타입

| 노드 타입 | 아이콘 | 설명 | 보상 |
|-----------|--------|------|------|
| **Start** | 시작 | 층의 시작점 | HP 일부 회복 |
| **Battle** | 검 | 일반 전투 | 골드 + 스킬 선택 |
| **Elite** | 해골 | 엘리트 전투 (강함) | 골드 + 레어 스킬 + 아이템 |
| **Boss** | 왕관 | 층 보스 | 대량 골드 + 유니크 보상 |
| **Event** | ? | 랜덤 이벤트 | 선택에 따른 보상/패널티 |
| **Shop** | 동전 | 상점 | 골드로 스킬/아이템 구매 |
| **Rest** | 캠프파이어 | 휴식 | HP 회복 또는 스킬 업그레이드 |

### 6.3 맵 생성 규칙

```
기본 규칙:
- 각 단계(layer)마다 2~4개 노드 생성
- 상하 단계 간 분기 연결 (한 노드는 최대 3개 하위 노드와 연결)
- 플레이어는 연결된 노드로만 이동 가능
- 보스 노드는 항상 마지막 단계에 1개

노드 분포 가중치 (층 1 기준):
- Battle: 50%
- Event: 20%
- Shop: 15%
- Rest: 15%
- Elite: 층당 최대 1개
```

### 6.4 데이터 구조

```
MapNode (순수 C#)
├── NodeType: 노드 타입 (enum)
├── Position: 층 내 위치 (int layer, int index)
├── Connections: 연결된 다음 노드 목록
├── IsVisited: 방문 여부
├── IsActive: 현재 선택 가능 여부
└── NodeData: 노드별 추가 데이터 (전투용 적 목록 등)

MapFloor (순수 C#)
├── FloorNumber: 층 번호
├── Layers: 단계별 노드 목록 (List<List<MapNode>>)
├── CurrentNode: 플레이어 현재 위치
└── IsCleared: 층 클리어 여부

MapGenerator (순수 C#)
├── GenerateFloor(floorNumber, config): 층 맵 생성
└── 규칙 기반 프록시럴 생성

GameRunState (순수 C#)
├── CurrentFloor: 현재 층
├── Gold: 보유 골드
├── PlayerParty: 파티 상태 (HP 유지, 스킬 인벤토리)
├── Inventory: 획득 아이템
└── RunHistory: 방문 노드 이력
```

### 6.5 플레이어 이동 규칙

```
1. 현재 노드에서 연결된 다음 단계 노드 중 선택
2. 선택한 노드로 이동 → 노드 이벤트 발생
3. 이벤트 완료 후 다음 선택지 활성화
4. 보스 노드 클리어 시 다음 층으로 이동 또는 게임 승리
```

### 6.6 기존 전투 시스템과의 연결

```
맵 노드(전투) 선택
    ↓
BattleScene 로드
    ↓ (적 데이터를 맵 노드에서 전달)
BattleSceneSetup.InitializeBattle(party, enemies)
    ↓
전투 완료
    ↓ (승리/패별 결과를 맵으로 반환)
RewardUI 표시 → 맵으로 복귀
```

- BattleSceneSetup이 외부에서 파티/적 데이터를 받을 수 있도록 API 확장
- 전투 결과(승리/패배)를 이벤트로 통지
- 파티 HP는 전투 간 유지 (GameRunState에서 관리)

## 7. 보상 시스템 상세 (Phase 3)

### 7.1 전투 보상

```
전투 승리 시:
┌─────────────────────────────┐
│  보상 카드 3장 중 1장 선택    │
│  ┌──────┐ ┌──────┐ ┌──────┐ │
│  │ 골드 │ │ 스킬 │ │ 골드 │ │
│  │ 30G  │ │ 강타 │ │ 50G  │ │
│  └──────┘ └──────┘ └──────┘ │
└─────────────────────────────┘

보상 풀 (일반 전투):
- 골드: 20~50 (가중치 40%)
- 일반 스킬: 기존 스킬과 다른 클래스 스킬 포함 (가중치 35%)
- 골드(대): 50~80 (가중치 15%)
- 아이템: 영구 버프 (가중치 10%)

보상 풀 (엘리트):
- 골드: 50~100
- 레어 스킬: 높은 위력/특수 효과
- 아이템: 100%

보상 풀 (보스):
- 골드: 100~200
- 유니크 스킬: 보스 전용 스킬 획득
- 캐릭터: 새 파티 멤버 (층 1 보스)
```

### 7.2 보상 데이터 구조

```
RewardData (ScriptableObject)
├── RewardType: 보상 타입 (Gold, Skill, Item, Character)
├── GoldAmount: 골드 금액 (Gold 타입일 때)
├── Skill: 스킬 데이터 참조 (Skill 타입일 때)
├── Item: 아이템 데이터 참조 (Item 타입일 때)
├── Character: 캐릭터 데이터 참조 (Character 타입일 때)
└── Rarity: 희귀도 (Common, Rare, Unique)
```

### 7.3 상점 시스템

```
상점 구성:
- 판매 슬롯: 5개 (스킬 3개 + 아이템 2개)
- 가격: 희귀도에 따라 30~150 골드
- 방문 시마다 랜덤 상품 생성
- 플레이어 스킬 판매 가능 (판매가 = 구매가의 50%)
```

### 7.4 아이템 시스템

```
ItemData (ScriptableObject)
├── ItemName: 아이템 이름
├── Description: 설명
├── ItemType: 타입 (PassiveBuff, Consumable, Relic)
├── EffectType: 효과 (MaxHPUp, ATKUp, DEFUp, HealPerTurn 등)
├── EffectValue: 효과 수치
├── Price: 상점 가격
└── Rarity: 희귀도
```

## 8. 이벤트 시스템 상세 (Phase 3)

### 8.1 이벤트 구조

```
EventData (ScriptableObject)
├── EventName: 이벤트 이름
├── Description: 상황 설명 (이야기 텍스트)
├── Choices: 선택지 목록 (2~3개)
│   ├── ChoiceText: 선택지 텍스트
│   ├── Outcome: 결과
│   │   ├── ResultText: 결과 설명 텍스트
│   │   ├── GoldChange: 골드 변화 (+/-)
│   │   ├── HPChange: HP 변화 (+/-, 파티 전체 또는 대상)
│   │   ├── GiveSkill: 스킬 획득 (선택)
│   │   ├── GiveItem: 아이템 획득 (선택)
│   │   └── ApplyStatus: 상태이상 부여 (선택)
│   └── Weight: 이 선택지의 가중치
└── EventType: 이벤트 분류 (Story, Treasure, Trap, NPC, Shrine)
```

### 8.2 이벤트 예시

| 이벤트명 | 설명 | 선택지 |
|----------|------|--------|
| 신비한 샘 | 빛나는 샘을 발견했다 | A: 마신다 (HP 30% 회복) / B: 떠난다 (골드 20 획득) |
| 상인의 제안 | 떠돌이 상인이 나타났다 | A: 스킬을 산다 (골드 40 소모, 랜덤 스킬) / B: 무시한다 |
| 함정 | 바닥이 무너졌다 | A: 점프한다 (50% 확률 피해 없음/HP 20% 피해) / B: 우회한다 (아무 일 없음) |
| 고대의 제단 | 신비한 제단이 있다 | A: 기도한다 (랜덤 버프) / B: 파괴한다 (골드 50, 랜덤 디버프) |
