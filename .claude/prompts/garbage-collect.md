# 가비지 컬렉터 프롬프트 템플릿

## 사용법
프로젝트에 오염이 누적되었다고 판단될 때, 또는 Phase 전환 시 실행하세요.

```
### 수집 타입
[경량 / 심층]

### 범위 (선택)
[전체 프로젝트 / 특정 폴더 / 특정 시스템]
```

---

## 경량 수집 (세션 종료 전)

매 작업 세션에서 수정한 파일에 대해 자동으로 실행.

### 체크리스트

1. **죽은 코드 스캔**
   - [ ] 이번에 수정한 파일에서 사용되지 않는 private 메서드/변수 있는지
   - [ ] 주석 처리된 코드 블록 있는지 (`// ...` 으로 비활성화된 로직)
   - [ ] 미사용 `using` 문 있는지
   - [ ] 빈 메서드 (구현 없이 `{}` 인 것) 있는지

2. **일관성 스캔**
   - [ ] 이번 변경이 CLAUDE.md 코딩 규칙과 일치하는지
   - [ ] 네임스페이스가 올바른지
   - [ ] 이벤트 기반 패턴을 위반하지 않았는지

3. **즉시 정화 항목** (사용자 승인 없이 제안)
   - 미사용 `using` 문 제거
   - 명백한 주석 처리된 코드 제거
   - 빈 catch 블록에 최소 로깅 추가

---

## 심층 수집 (Phase 전환 / 사용자 요청 시)

전체 프로젝트를 스캔하여 누적된 오염 제거.

### Phase 1: 스캔

#### 1.1 죽은 코드 탐지
```
각 .cs 파일에 대해:
- private 메서드/변수 중 프로젝트 내 참조가 없는 것
- internal 클래스 중 인스턴스화되지 않는 것
- enum 값 중 사용되지 않는 것
- 이벤트 중 구독자가 없는 것
```

#### 1.2 미사용 에셋 탐지
```
Assets/03.Data/의 각 ScriptableObject에 대해:
- 씬 파일(.unity)에서 참조되는지
- 스크립트에서 로드/참조되는지
- 다른 ScriptableObject에서 참조되는지

Assets/03.Data/Prefabs/의 각 프리팹에 대해:
- 씬 또는 스크립트에서 Instantiate 되는지
```

#### 1.3 기획 일치성 검사
```
GameDesign.md 기준:
- 현재 구현된 턴 사이클이 기획과 일치하는지
- 캐릭터 클래스/스탯이 기획 범위 내인지
- 스킬 구조가 기획과 일치하는지
- 보상/진행 시스템이 기획과 충돌하는지
```

#### 1.4 아키텍처 위반 탐지
```
CLAUDE.md 규칙 기준:
- UI 스크립트에 게임 로직이 있는지
- MonoBehaviour가 불필요하게 사용되었는지
- FindObjectOfType/Find/PlayerPrefs 사용이 있는지
- 하드코딩된 매직넘버가 있는지
- 이벤트를 거치지 않은 직접 참조가 있는지
```

#### 1.5 중복 로직 탐지
```
동일하거나 유사한 코드 패턴:
- 데미지 계산 로직이 여러 곳에 있는지
- HP 업데이트 로직이 중복되는지
- UI 업데이트 로직이 여러 클래스에 흩어져 있는지
```

#### 1.6 유령 참조 탐지
```
- 씬 파일에서 존재하지 않는 스크립트 참조 (Missing Script)
- ScriptableObject 필드에서 누락된 참조 (None/None (Missing))
- 프리팹의 깨진 참조
```

#### 1.7 스텁/TODO 방치 탐지
```
- TODO/HACK/FIXME/TEMP 주석이 있는 위치
- 메서드 본문이 throw NotImplementedException인 것
- // TODO: 로 시작하며 2주 이상 방치된 항목
```

### Phase 2: 분류 및 보고

발견된 각 항목을 분류하여 보고서 작성:

```
## 가비지 컬렉터 보고서

### 즉시 삭제 가능 (N건)
| 파일 | 항목 | 사유 |
|------|------|------|
| ... | 미사용 using문 | 참조 없음 |
| ... | 주석 처리된 코드 | 기능 비활성화 상태 |

### 확인 후 삭제 권장 (N건)
| 파일 | 항목 | 사유 | 위험도 |
|------|------|------|--------|
| ... | 미사용 ScriptableObject | 참조되지 않음 | 낮음 |
| ... | 중복 데미지 계산 | TurnManager와 UI에 중복 | 중간 |

### 보류 (N건)
| 파일 | 항목 | 사유 | 보류 이유 |
|------|------|------|-----------|
| ... | MPBarUI 스텁 | 미구현 | Phase 3에서 사용 예정 |

### 아키텍처 위반 (N건)
| 파일 | 위반 내용 | 권장 조치 |
|------|-----------|-----------|
| ... | UI에서 데미지 계산 | 로직을 Combat 계층으로 이동 |
```

### Phase 3: 정화 (사용자 승인 후)

1. **삭제 순서**:
   ```
   데이터 에셋 (03.Data/) → 스크립트 (02.Scripts/) → 씬 (01.Scenes/)
   ```
   하위 의존성부터 삭제하여 깨진 참조 최소화

2. **삭제 방법**:
   - 코드: Edit 도구로 해당 라인/블록 제거
   - 에셋: `rm` 명령으로 파일 삭제 (Unity가 재시작 시 .meta 자동 정리)
   - 씬 참조: Unity 에디터에서 수동 정리 필요 시 안내

3. **삭제 후 필수 검증**:
   ```
   [ ] Unity 콘솔 에러 0건 확인
   [ ] 컴파일 성공 확인
   [ ] 기존 씬 로드 정상 확인
   [ ] git diff로 삭제 범위 최종 확인
   ```
