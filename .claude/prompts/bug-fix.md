# 버그 수정 프롬프트 템플릿

## 사용법
버그를 발견했을 때 아래 형식으로 AI에게 요청하세요.

---

## 입력 양식

```
### 버그 설명
[무엇이 어떻게 잘못되는지 간단히 설명]

### 재현 단계
1. [단계 1]
2. [단계 2]
3. [단계 3]

### 기대 동작
[정상적으로 동작해야 하는 모습]

### 실제 동작
[현재 발생하는 문제]

### 관련 파일 (선택)
[의심되는 스크립트 경로]
```

## AI 작업 프로세스

### Step 1: 원인 파악
1. 관련 코드 전체 읽기 (해당 메서드, 호출 체인)
2. Unity 콘솔 에러/경고 확인 (`get_console_logs`)
3. 씬 상태 확인 (해당 GameObject, 컴포넌트)
4. 직전 변경 사항 확인 (`git diff`)

### Step 2: 수정 원칙
- 최소한의 변경으로 수정 (광범위 리팩토링 금지)
- 기존 public API 시그니처 유지
- 수정이 다른 시스템에 미치는 영향 분석
- 상태이상/드로우 시스템 수정 시 EffectProcessor, SkillDrawSystem 연쇄 영향 확인

### Step 3: 검증
- 수정 후 Unity 콘솔 에러 확인
- 관련 시스템 정상 동작 확인
- 엣지 케이스 고려 (HP 0 이하, 빈 타겟, 중복 상태이상 등)

## 체크리스트

- [ ] 네임스페이스 규칙을 지켰는가?
- [ ] 이벤트 기반 패턴을 유지했는가?
