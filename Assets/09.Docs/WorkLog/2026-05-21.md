## 세션 1 — Battle↔Map 런 상태 유지 및 Battle→Reward 연결

### GameRunState 싱글톤화
- **상태**: 완료
- **관련 파일**: `Assets/02.Scripts/Map/GameRunState.cs`
- **내용**:
  - 기존: 매 씬마다 새 인스턴스 생성 → 씬 전환 시 런 상태 초기화됨
  - `static Instance` 싱글톤으로 변경, 씬 전환 시에도 런 상태 유지
  - `Create()` / `Destroy()` / `Cleanup()` 정적 메서드 추가
  - 생성자 `private`화로 외부 new 금지
  - `PeekRandomSkill()` / `PeekRandomItem()` 추가 (보상 미리보기용, 실제 획득은 ClaimReward에서)
- **결과**: Battle→Map 복귀 시 골드, 파티 HP, 층 진행도 유지

### BattleResult 전투 결과 전달 시스템
- **상태**: 완료
- **관련 파일**: `Assets/02.Scripts/Combat/BattleResult.cs` (신규)
- **내용**:
  - 정적 클래스로 전투 결과를 씬 전환 간 전달
  - `SetBattleType(MapNodeType)`: MapSceneSetup에서 전투 시작 전 노드 타입 설정
  - `SetResult(bool victory)`: BattleSceneSetup에서 전투 종료 시 결과 기록
  - `Clear()`: MapSceneSetup에서 보상 처리 완료 후 초기화
- **결과**: 전투 승리/패배 결과가 MapScene에 정상 전달

### MapSceneSetup 씬 복귀 및 보상 흐름
- **상태**: 완료
- **관련 파일**: `Assets/02.Scripts/UI/Map/MapSceneSetup.cs`
- **내용**:
  - `RestoreExistingRun()`: 씬 복귀 시 기존 런 인스턴스 복원, 이벤트 재구독, MapView 복원
  - `HandleBattleResult()`: 승리 시 `OnBattleVictory()` → `_rewardUI.ShowRewards()`, 패배 시 `EndRunDefeat()`
  - `InitializeSubUIs()`: 서브 UI 초기화 로직 공통 추출
  - 보스 클리어 → 다음 층 이동을 보상 선택 이후(`OnRewardComplete`)로 이동
  - `StartBattle()`에 `BattleResult.SetBattleType()` 추가
- **결과**: Map→Battle→승리→Reward→Map 전체 사이클 동작

### RewardManager 지연 획득 패턴
- **상태**: 완료
- **관련 파일**: `Assets/02.Scripts/Reward/RewardManager.cs`
- **내용**:
  - `GenerateRewards()`: `AcquireRandomSkill/Item` → `PeekRandomSkill/Item` 변경 (미리보기만)
  - `ClaimReward()`: 선택 시점에 `runState.AcquireSkill/Item` 실제 획득
  - 기존: 보상 생성 시 즉시 획득 → 취소 불가
  - 변경: 보상 미리보기 → 플레이어 선택 시 획득 (지연 획득)
- **결과**: 보상 선택 전 인벤토리에 추가되지 않음
