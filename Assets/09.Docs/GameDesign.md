# Team Log - Game Design Document

## 1. 게임 개요

| 항목 | 내용 |
|------|------|
| **장르** | 턴제 전투 로그라이크 |
| **플랫폼** | PC (Unity) |
| **플레이어** | 1인 (4인 파티 조작) |
| **핵심 컨셉** | 스킬 선택이 아닌 랜덤 스킬 드로우 |

### 핵심 아이디어
- 매 턴, 각 캐릭터는 보유한 4개 스킬 중 **1개를 랜덤**으로 뽑아 사용
- 플레이어는 "어떤 스킬을 쓸지"가 아니라 **"랜덤으로 주어진 스킬을 어떻게 조합할지"** 선택
- 전략의 핵심: 상황에 맞는 캐릭터 순서 배치 + 스킬 타겟팅

---

## 2. 전투 시스템

### 2.1 턴 구조

```
[전투 시작]
    ↓
[턴 시작]
    ↓
[드로우 페이즈] → 각 캐릭터별로 스킬 1개 랜덤 드로우
    ↓
[액션 페이즈] → 플레이어가 순서/타겟 지정
    ↓
[실행 페이즈] → 지정된 순서대로 스킬 실행
    ↓
[적 턴] → 적 행동
    ↓
[턴 종료] → 다음 턴 시작
```

### 2.2 스킬 드로우 시스템

```
캐릭터당 스킬 슬롯: 4개
    ↓
턴 시작 시 각 캐릭터마다 1개 스킬 랜덤 선택
    ↓
플레이어에게 4개의 스킬 카드 제공 (캐릭터별 1개씩)
    ↓
플레이어가 실행 순서와 타겟 지정
```

## 3. 캐릭터 시스템

### 3.1 기본 스탯

| 스탯 | 설명 |
|------|------|
| HP | 체력 |
| ATK | 공격력 |
| DEF | 방어력 |

### 3.2 역할 클래스 (예시)

| 클래스 | 특징 | 스킬 예시 |
|--------|------|-----------|
| **전사** | 높은 HP, 근접 딜러 | 강타, 방패, 도발, 분노 |
| **마법사** | 높은 ATK, 낮은 HP | 파이어볼, 얼음창, 마나방어, 메테오 |
| **힐러** | 치유, 버프 | 치유, 보호막, 정화, 부활 |
| **도적** | 높은 SPD, 치명타 | 급소공격, 은신, 독바르기, 연막 |

### 3.3 스킬 구조

```
Skill
├── Name: 스킬 이름
├── Description: 설명
├── Type: 공격/치유/버프/디버프
├── TargetType: 단일/전체/아군/자신
├── Power: 위력 (기본 수치)
├── Cost: 비용 (있다면)
├── Effects: 추가 효과 (상태이상 등)
└── Weight: 드로우 가중치
```

## 4. 적 시스템

### 4.1 적 행동 패턴

- **패턴 기반**: 미리 정해진 행동 순서
- **의도 표시**: 다음 턴에 할 행동을 플레이어에게 미리 공개
  - 예: "공격 준비", "버프", "강력한 공격 충전"

### 4.2 적 타입

| 타입 | 특징 |
|------|------|
| 일반 | 기본 스탯, 단순 패턴 |
| 엘리트 | 높은 스탯, 특수 능력 |
| 보스 | 다중 페이즈, 특수 메커니즘 |

---

## 5. 진행 구조

### 5.1 맵 진행

```
[시작] → [일반 전투] → [이벤트] → [일반 전투] → [상점] → [엘리트] → ... → [보스] → [다음 층]
```

### 5.2 보상 시스템

| 보상 타입 | 설명 |
|-----------|------|
| 골드 | 상점에서 사용 |
| 새 스킬 | 캐릭터에 스킬 추가 |
| 스킬 강화 | 기존 스킬 업그레이드 |
| 아이템 | 영구 버프/장비 |
| 캐릭터 | 새 파티 멤버 |

## 7. 개발 우선순위 (제안)

1. 기본 데이터 구조 (Character, Skill)

4. 기본 UI

2. 적 AI
3. 전투 연출

### Phase 3: 로그라이크 요소
1. 맵 시스템
2. 보상/상점
3. 이벤트

### Phase 4: 폴리싱
1. 사운드
2. 이펙트
3. 밸런싱
